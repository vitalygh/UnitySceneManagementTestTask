Используя игровой движок Unity, необходимо сделать приложение с "3D" графикой (у камеры используется перспективная проекционная матрица).

В этом приложении можно будет управлять камерой (вращать и перемещать) и создавать кубы и шары на горизонтальной плоскости (далее "земля"), Y которой равен 0 (т.е. (A = 0, B = 1, C = 0, D = 0)), перемешать объекты, а также сохранять сцену и загружать её состояние. 

Запуск приложения не должен требовать наличие редактора Unity, т.е. приложение должно быть "собрано".

1. Сохранения
* 1.0. Сохраняться должны созданные объекты и 2 поля в UI (пункты 3.0. и 3.1.)
* 1.1. Одного сохранения достаточно. Новое сохранение перезаписывает предыдущее.

2. Камера
* 2.0. Перемещение камеры "вперед", "назад", "влево", "вправо" клавишами клавиатуры W, S, A, D
* 2.1. Поворот камеры при помощи "мыши" по двум осям (Yaw, Pitch) и только если зажата Правая кнопка мыши. 
* 2.2. Наклон вверх-вниз (Pitch) должен ограничиваться в диапазоне +90/-90 градусов (от изначальной ориентации, смотрящей "вперед").
* 2.3. Камера не должна уметь опускаться ниже Y = 0 (сквозь "землю")
	
3. UI в screen space-е с таким функционалом:
* 3.0. Текстовое поле, принимающее множитель к чувствительности мыши (floating point число, по умолчанию значение 1.0)
* 3.1. Текстовое поле, принимающее множитель к скорости перемещения камеры (floating point число, по умолчанию значение 1.0)
* 3.2. ComboBox (Dropdown) с выбором типа создаваемого объекта, всего два варианта:  Cube, Sphere
* 3.3. Кнопка очистки сцены ("Clear scene"), удаляющая со сцены все созданные объекты.
* 3.4. Кнопка сохранения сцены ("Save scene") - сохраняет текущее состояние сцены куда-то так, что бы оно могло быть загружено при следующем запуске приложения.
* 3.5. Кнопка загрузки сцены ("Load scene") - заменяет текущее состояние сцены на загруженное из сохранения.
	
4. Создание/Удаление объектов:
* 4.0. Созданный объект должен ставиться на "землю" вплотную, не пересекая её.
* 4.1. Ctrl + LeftMouseButton клик создаёт на "земле" в месте клика либо куб, либо шар в зависимости от того, что выбрано в UI (пункт 3.2.)
* 4.2. Alt + LeftMouseButton клик по существующему объекту удаляет его из сцены.
	
5. Редактирование положения объектов:
* 5.1. Если по объекту произведён клик левой кнопкой мыши (без Ctrl и Alt), то этот объект должен постоянно перемещаться в место пересечения плоскости и курсора мыши пока левую кнопку мыши не отпустит пользователь.
